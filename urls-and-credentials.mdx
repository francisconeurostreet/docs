Your onboarding contact will provide the specific values you need to access Ignite.

Use placeholders in these docs until you receive your real values:

- `{slug}`: your firm identifier.
- `{BASE_URL}`: your firm host.

Your `{BASE_URL}` may be different from your `{slug}`. Use the exact host provided during onboarding.

Do not include a path in `{BASE_URL}`. Dashboard pages and API endpoints use the same host with different paths.

## URLs

| Resource | URL |
| --- | --- |
| Firm host | `{BASE_URL}` |
| Dashboard pages | Root-level paths, such as `{BASE_URL}/tickblaze-manager/users` or `{BASE_URL}/challenges` |
| API endpoints | Usually prefixed with `/api/`, such as `{BASE_URL}/api/accounts/summary/` |
| Login endpoint | `{BASE_URL}/auth/login/` |
| API documentation | This Ignite API documentation site |
| SQS queue | Provided only if SQS is enabled for your firm |

## Credentials

Dashboard and API credentials are provided separately during onboarding.

<Tip>
  After you confirm your `{BASE_URL}` and credentials, use the [quickstart](/quickstart) to authenticate and make your first API request.
</Tip>
