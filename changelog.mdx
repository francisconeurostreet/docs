---
title: "Changelog"
description: "Customer-facing product and API updates."
---

<Update label="API" description="June 5, 2026">
  Added new account summary fields to help customers inspect account status, balance, equity, and challenge progress from one response.
</Update>

<Update label="Fix" description="June 4, 2026">
  Fixed an issue where some account activity reports could show stale position counts after recent trading activity.
</Update>

<Update label="API" description="June 3, 2026">
  Added filtering support for trading history by opened and closed timestamps. This makes it easier to export activity for a specific reporting window.
</Update>

<Update label="Improvement" description="June 2, 2026">
  Improved entitlement package visibility so customers can more clearly review which market data packages are active for a user.
</Update>

<Update label="Fix" description="May 30, 2026">
  Fixed a billing export inconsistency where some rows could use a different authorization timestamp than the customer-facing entitlement view.
</Update>

<Update label="Deprecation" description="May 28, 2026">
  The legacy withdrawal status field will remain available during the transition period, but customers should begin using the normalized status field in new integrations.
</Update>

<Update label="Breaking Change" description="May 20, 2026">
  Preview example only: a previously optional request field is now required for new integrations. Existing integrations continue to work during the compatibility window.
</Update>
